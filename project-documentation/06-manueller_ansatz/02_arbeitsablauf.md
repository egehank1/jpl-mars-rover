# Arbeitsablauf

Die Projektarbeit erfolgte **iterativ und parallelisiert**.

## Organisation
- Arbeit auf **eigenen Rechnern**
- keine zentrale Entwicklungsumgebung
- Abstimmung über regelmäßige Meetings

## Vorgehen
- schnelle Prototypen statt komplexer Simulation
- frühes Testen einzelner Komponenten
- schrittweise Integration in bestehende ROS2-Struktur

Besonderer Fokus lag auf dem **AutoFOCUS-orientierten Ansatz**,
da unterschiedliche Betriebssysteme, ROS-Versionen und Tool-Versionen
einen großen Einfluss auf die Ergebnisse hatten.
